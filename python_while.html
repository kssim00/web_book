1. While 문의 기본 구조

while은 '~하는 동안'이라는 뜻입니다. 조건이 맞으면 계속 실행하고, 틀리면 즉시 종료합니다.

⚠️ while문을 쓰기 전에 반드시 해야 하는 것!

while문은 3가지를 항상 직접 써줘야 합니다. 하나라도 빠지면 오류가 나거나 무한루프가 됩니다.

① 시작값

i = 1

반복을 어디서 시작할지

② 조건식

while i <= 3:

언제까지 반복할지

③ 증가식

i += 1

매 반복마다 변수를 바꿔줌

❌ ③ 증가식을 빠뜨리면? → i가 계속 1이라서 조건이 영원히 True → 무한루프 발생!
❌ ① 시작값을 안 쓰면? → i가 뭔지 몰라서 → 오류(NameError) 발생!

💡 시작값을 0으로 할까, 1로 할까?

i = 0 으로 시작할 때

i = 0
while i < 3:
    i += 1
    print(i)
# 출력: 1, 2, 3

i = 1 로 시작할 때

i = 1
while i <= 3:
    print(i)
    i += 1
# 출력: 1, 2, 3

👉 결과는 같지만 시작값이 다르면 조건식(< vs <=)도 함께 바뀌어야 해요. 어느 쪽이든 괜찮지만 일관성 있게 쓰는 습관이 중요합니다!

i = 0                 # ① 시작값 선언 (필수!)

while i < 3:        # ② 조건식 (참일 동안 반복)
    i = i + 1         # ③ 증가식 (없으면 무한루프!)
    print(f"현재 숫자는 {i}")
현재 숫자는 1
현재 숫자는 2
현재 숫자는 3

1부터 5까지 출력하는 while 루프입니다. 세 개의 빈칸을 채워보세요!

2. 데이터의 끝을 알리는 '보초값(Sentinel)'

🤔 왜 보초값이 필요할까요?

반복 횟수를 아는 경우

"1부터 10까지 출력해" → 횟수가 정해져 있어서 while 조건으로 바로 제어 가능

반복 횟수를 모르는 경우 ← 보초값 필요!

"점수를 계속 입력받아" → 몇 개 입력할지 모름. 특별한 값을 입력하면 멈추도록 약속해야 함

📌 보초값 선택의 핵심 규칙

보초값은 실제 데이터로 절대 나올 수 없는 값으로 정해야 합니다.

점수 입력 (0~100점)

❌ 0, 50, 100 — 실제 점수와 겹침
✅ -1 — 점수로 나올 수 없는 값

주사위 눈 입력 (1~6)

❌ 1, 3, 6 — 실제 눈과 겹침
✅ 0 또는 -1 — 주사위에 없는 값

💻 코드로 보기 — 점수를 입력받다가 -1이 오면 멈추기

score = int(input("점수 입력(-1 종료): "))   # 첫 입력

while score != -1:                          # -1이 아닌 동안 반복
    print("입력된 점수:", score)
    score = int(input("점수 입력(-1 종료): ")) # 다음 입력

print("입력 종료!")
💡 왜 while 전에 먼저 input을 받나요?
while 조건에서 score를 비교하려면 score가 이미 있어야 해요. 처음부터 값이 없으면 오류가 납니다.
점수를 입력하고, 멈추고 싶을 때 -1을 입력하세요.

🎯 [상황 판단] 올바른 보초값 고르기

상황 ① 학생 시험 점수(0~100점) 입력 — 종료 신호로 적합한 값은?
상황 ② 주사위 굴리기(1~6) 입력 — 보초값으로 쓰면 안 되는 값은?

3. 흐름 제어: Break vs Continue

🚪 Break

반복문을 완전히 끝냅니다. 조건과 상관없이 즉시 탈출합니다.

결과: 1 2
⏭️ Continue

이번 회만 건너뛰고, 다음 조건 확인으로 돌아갑니다.

결과: 1 2 4 5

3을 만나는 순간 반복이 완전히 끝납니다. 4, 5는 출력 안 돼요.

i = 1
while i <= 5:
    if i == 3:
        break
    print(i)
    i += 1
print("반복 끝")

3만 건너뛰고 반복은 계속됩니다. 4, 5는 정상 출력돼요.

i = 0
while i < 5:
    i += 1
    if i == 3:
        continue
    print(i)
print("반복 끝")

📊 한눈에 비교 — i == 3일 때 무슨 일이 생길까?

키워드 i == 3 일 때 출력 결과
break 반복문 자체를 완전히 끝냄 1 2 반복 끝
continue 3만 건너뛰고 반복은 계속 1 2 4 5 반복 끝

🤔 [상황 판단] Break일까, Continue일까?

상황 ① 비밀번호가 맞으면 반복문을 완전히 끝낸다
상황 ② 짝수만 출력하고, 홀수는 건너뛴다

4. while — else

while문은 else와 짝을 이룰 수 있습니다. 조건이 False가 되어 정상 종료되면 else 블록이 실행됩니다. (break로 탈출하면 실행되지 않습니다!)

count = 0
while count < 3:
    print(count, '은 3보다 작은 수')
    count = count + 1
else:
    print(count, '이다! stop!!')

🎯 while-else 출력 예측하기

while n > 0:

5. 무한 루프 (Infinite Loop)

🤔 무한루프, 나쁜 건가요?

❌ 실수로 생긴 무한루프

증가식을 빠뜨리거나 조건을 잘못 써서 프로그램이 멈추지 않는 것 → 버그!

✅ 의도적인 무한루프

while True: 로 일부러 무한 반복을 만들고, 원하는 조건이 되면 break로 탈출 → 정상적인 패턴!

👉 몇 번 반복할지 미리 알 수 없을 때 (비밀번호가 몇 번 만에 맞을지, 언제 입력이 끝날지) 의도적으로 씁니다.

📌 while 조건식 vs while True — 언제 어떤 걸 쓸까?

while 조건식

끝나는 시점이 처음부터 정해진 경우

i = 1
while i <= 10:
    print(i)
    i += 1

while True + break

끝나는 시점을 실행 중에 판단하는 경우

while True:
    pw = input()
    if pw == "1234":
        break
💡 한 줄 기준: 끝나는 조건을 미리 알면 → while 조건식 / 실행 중에 결정되면 → while True + break

💻 코드로 보기 — 암호가 맞을 때까지 계속 입력받기

while True:
    code = input("암호를 입력하세요: ")
    if code == "1234":
        print("정답! 탈출!")
        break              # 맞으면 반복 종료
    print("틀렸습니다. 다시 입력하세요.")
암호를 입력하세요. "1234"를 입력하면 탈출합니다.

🐛 [버그 찾기] 이 코드의 문제점은?

아래 코드는 올바르게 작동하지 않습니다. 이유가 무엇일까요?

while True:
    pw = input("비밀번호: ")
    print("완료")
